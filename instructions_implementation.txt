Project Implementation Plan: Phases, Sub-Phases & Detailed Steps
----------------------------------------------------------------


Phase 1: Foundation & Bootstrapping
-----------------------------------
DONOT DEVIATE FROM THE ALREADY CREATED DIRECTORY STRUCTURE
Incorporates every bit of logic from the full user-flow and splits into bite-sized work items.
Goal: Get a runnable CLI scaffold, configuration, logging, and DI in place.

1. Repository & Environment
   - Initialize Git repo, add `.gitignore`.
   - Create Python 3.10+ virtualenv, activate.
   - Add `requirements.txt`: `twilio`, `rich`, `requests`, `python-dotenv`, `click`, `pytest`.

2. Configuration & Secrets
   - `app/gateways/config.py`: load `.env` via `dotenv`; expose `TWILIO_ACCOUNT_SID`, `AUTH_TOKEN`, etc.
   - `shared/settings.py`: dataclass for config values.

3. Logging
   - `shared/logging.py`: configure root logger (DEBUG/INFO), console handler, rotating file handler.
   - Use `logger = logging.getLogger(__name__)` in every module.

4. Dependency Injection
   - `app/interfaces/cli_controller.py`: define `CLIController` wiring `services` and `menus`.
   - Expose `main()` → `CLIController().run()`.

5. Entry Point
   - `__main__.py` calls `main()`.
   - Smoke-run: verify “Main Menu” renders.

Phase 4: Gateway Layer & Services
---------------------------------
DONOT DEVIATE FROM THE ALREADY CREATED DIRECTORY STRUCTURE
Incorporates every bit of logic from the full user-flow and splits into bite-sized work items.
Goal: Wrap HTTP, Twilio SDK, and file I/O behind clean interfaces.

1. Twilio Gateway (`app/gateways/twilio_gateway.py`)
   - `get_client()` → `Client`.
   - Methods: `list_numbers(filters)`, `purchase_number(sid)`, `release_number(sid)`, `update_number_config(sid, cfg)`, `make_call(from_, to)`, `send_sms(from_, to, body)`, `list_logs(type, sid, page_token)`.

2. HTTP Gateway (`app/gateways/http_gateway.py`)
   - Twilio REST “AvailablePhoneNumbers” raw calls.
   - `search_batch(country, type, capabilities, page_size, page_token)`.

3. File Logger (`app/gateways/file_logger.py`)
   - JSON debug logger (optional).

4. Service Layer (`app/services/`)
   - NumberService: `search_available(...)`, `purchase_numbers(...)`, `list_active_numbers()`, `release_number(sid)`, `get_number_config(sid)`, `update_number_config(sid, cfg)`.
   - VoiceService: `make_call(from_, to)`.
   - MessagingService: `send_sms(from_, to, body)`.
   - AccountService: `get_usage()`, `get_billing()`.
   - LogsService: `list_logs(type, sid, page)`.

Phase 5: Purchase Numbers Flow
------------------------------
DONOT DEVIATE FROM THE ALREADY CREATED DIRECTORY STRUCTURE
Incorporates every bit of logic from the full user-flow and splits into bite-sized work items.
Goal: Fully implement nested “1. Purchase Numbers” guided flow.

1. SelectCountryMenu
   - US/CA/GB/AU or “Other” free-form ISO.

2. SelectTypeMenu
   - Fetch types via `twilio_gateway.list_numbers(country_iso)`.

3. SearchModeMenu
   - ByDigitsMenu: 3–10 digits; US & len=3 → lookup area codes; else “contains”.
   - ByLocalityMenu: fetch distinct localities via HTTP gateway metadata.

4. SelectCapabilitiesMenu
   - Choose Voice, SMS, MMS (at least one).

5. SearchExecutionMenu
   - “Searching…” panel; call `NumberService.search_available()`.
   - Loop 1 s, batch of 50, dedupe, update Rich progress bar every 50.
   - Stop at 500 unique or 3 empty rounds.

6. SearchResultsMenu
   - Paginated table of `NumberRecord`.
   - Keys: [n]/[p], [s], [j]/[c], [x], [b].

7. PurchaseConfirmMenu
   - Confirm queue vs immediate purchase.
   - Call `NumberService.purchase_numbers()`.
   - Show ✔/✖ per number.
   - Pause, then back.
